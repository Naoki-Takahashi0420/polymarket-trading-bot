# AI Trading Bot — 要件定義

## プロジェクト概要
日本株（東証）向けの自動売買ボット。kabu Station API を使用。
社内専用の「秘密のキャッシュマシーン」として運用する。

## Phase 1: レンジ取引ボット（MVP）

### 目的
レンジ相場の銘柄を自動検出し、下限で買い→上限で売りを繰り返す。
感情を完全に排除した淡々とした取引。

### 機能要件

#### 1. データ取得（Data Fetcher）
- Yahoo Finance Japan から日本株の過去データを取得（yfinance）
- 日足・5分足データに対応
- 東証上場銘柄をカバー
- J-Quants API 対応（将来Phase）

#### 2. レンジ銘柄検出（Range Detector）
- 過去N日間（デフォルト60日）の価格データを分析
- レンジ判定基準:
  - ボリンジャーバンド幅が一定以下
  - ATR（Average True Range）が一定以下
  - 価格が一定の上限・下限バンド内に収まっている割合
- 候補銘柄のスコアリングとランキング
- NTTのような大型株を優先（流動性確保）

#### 3. バックテスト基盤（Backtester）
- backtesting.py ベースのバックテストエンジン
- レンジ取引戦略のバックテスト
- 結果出力: 勝率、PnL、最大ドローダウン、シャープレシオ
- HTML レポート生成

#### 4. 売買シグナル生成（Signal Generator）
- レンジ下限到達 → 買いシグナル
- レンジ上限到達 → 売りシグナル
- 損切りライン設定（レンジ下限のN%下）
- ポジションサイジング（Kelly Criterion）

#### 5. kabu Station API 接続（Executor）
- REST API でトークン取得・認証
- 銘柄情報取得
- 注文発注（指値注文）
- ポジション管理
- ペーパートレードモード対応

#### 6. 通知（Notifier）
- Discord Webhook で売買シグナル・約定通知
- 日次レポート送信

### 非機能要件
- Python 3.11+
- 東証取引時間（9:00-15:30 JST）のみ稼働
- エラーハンドリング・リトライ機構
- ログ出力（ファイル + コンソール）
- 設定ファイル（YAML）で銘柄・パラメータ管理

### 技術スタック
- Python 3.11+
- yfinance（データ取得）
- backtesting.py（バックテスト）
- pandas / numpy（データ処理）
- requests（API通信）
- pyyaml（設定管理）
- Discord Webhook（通知）
