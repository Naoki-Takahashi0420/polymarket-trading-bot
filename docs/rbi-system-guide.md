## 1. はじめに

この動画は、Claude Code（AI）を活用してPolymarket（予測市場）およびその他の取引所向けの自動売買ボットを構築する方法を、ステップバイステップで解説するライブ配信形式の教材である。

発表者は「**RBIシステム**」と名付けた3段階のフレームワーク（Research → Backtest → Incubate）を提唱し、AIの力を借りてコーディング知識がなくても自動売買を実現できることを主張している。

動画の後半では有料コース（Algo Trade Camp）への誘導も含まれるが、前半〜中盤で紹介される手法やフレームワーク自体は無料のリソースでも再現可能な内容となっている。

## 3. なぜ自動売買なのか - 手動トレードの限界

### 3.1 扁桃体ハイジャック（Amygdala Hijack）

発表者は、手動トレードが失敗する科学的根拠として「扁桃体ハイジャック」を紹介している。

| 脳の部位 | 処理速度 | 役割 |
|---------|---------|-----|
| 扁桃体（Amygdala）= 感情脳 | **12ミリ秒** | 脅威を感知し、即座に反応する |
| 前頭前皮質（Prefrontal Cortex）= 論理脳 | **500ミリ秒** | 状況を分析し、合理的判断を下す |

- 感情は論理の**40倍速く**発火する
- 進化的には有利（蛇を見たら即座に跳ぶ）だが、トレードでは致命的
- 赤いローソク足や損失ポジションを見ると、論理脳が追いつく前にパニック売りしてしまう
- スタンフォード大学の教授の発言：「感情的な状態では、論理的に思考することは文字通り不可能」

### 3.2 取引所の設計

- 取引所はカジノのように設計されている（赤・緑のフラッシュ、レバレッジの誘惑）
- ウォール街はユーザーのポジションを閲覧可能（何十年も前から）
- 「自動売買はエリート校出身者だけのもの」という障壁をAIが破壊した

### 3.3 解決策

> **「ボットには扁桃体がない」**

事前定義されたエントリー・エグジット・ポジションサイジングが、感情を取引から完全に排除する。

## 4. RBIシステム - 自動売買の3ステップ

発表者が提唱する取引自動化プロセスの体系。

```
R → Research（リサーチ）    : 利益の出る戦略を見つける
B → Backtest（バックテスト）  : 過去のデータで戦略が機能するか検証する
I → Incubate（インキュベーション）: 小額でボットを稼働させて実戦テストする
```

このサイクルを繰り返し、検証済みの戦略のみをスケールアップしていく。

## 6. Backtest（バックテスト）- 過去データで検証

### 6.1 バックテストとは

過去のデータを使って、戦略が実際に利益を出したかどうかを検証するプロセス。

### 6.2 Claude AIによるバックテストの自動化

1. バックテスト用テンプレートコード（`backtesting.py`）をClaude AIに参照させる
2. 過去の価格データ（例：ビットコインの5分足データ）を読み込ませる
3. テストしたい戦略の概要をプロンプトとして送る
4. **複数のエージェントを並列起動**して、同時に複数の戦略をテスト

### 6.3 実際のバックテスト結果

**成功した戦略:**

| 戦略名 | 勝率 | エッジ | 推定P&L |
|-------|------|-------|--------|
| MACD 3-15-3（5分足） | 60% | 0.618 | +$119,000 |
| 戦略B | 59% | - | - |
| 戦略C | 63% | - | - |

**失敗した戦略（事前に排除できた）:**

| 戦略名 | リターン |
|-------|--------|
| Capitulation Reversion | **-94%** |
| Capitulation + RSI Oversold | **-94%** |
| Capitulation + Bollinger Band Squeeze | **-94%** |

> 発表者コメント：「多くの人がこのような-94%リターンの戦略を "勝てる" と思い込んで1年間トレードし続けている。バックテストなしでは、その事実に気づくことすらできない」

### 6.4 効率化の成果

- 以前：バックテストに**8時間以上**
- 現在（Claude AI使用）：**約1時間**で3つの戦略を並列テスト

### 6.5 データの入手方法

- 暗号通貨データ：発表者のロードマップ上にダウンロードリンクあり
- Coinbaseの1分足データ：Polymarketのバックテストにも使用可能
- 通常は数百ドルかかるデータも、適切な方法で無料取得が可能

## 7. Incubate（インキュベーション）- 小額実戦テスト

### 7.1 インキュベーションの原則

- バックテストで有望な戦略を発見しても、**いきなり大金を投入しない**
- 最小額（Polymarketなら$1、HyperLiquidなら$10）で開始
- 最低**数週間〜1ヶ月**のインキュベーション期間を設ける

### 7.2 スケールアップの段階

```
$1〜$10 → $20 → $50 → $100 → 検証結果に応じてさらに拡大
```

### 7.3 発表者の失敗談

> 「バックテストせずに$10,000をボットに投入し、全額失った。戦略が機能するか確認せずに自動化した結果だった」

### 7.4 インキュベーション中の注意

- 数時間ごとにボットの動作を確認する
- バグがあれば修正する
- ボットを画面上に常に表示しておき、エラーを早期発見する
- 「すぐに大きく稼ぎたい」という誘惑に負けない

## 8. Claude Codeを使ったボット構築

### 8.1 基本的なワークフロー

1. Claude Code（Opus 4.6）を起動
2. 既存のボットコードを**テンプレート**としてAIに参照させる
3. 新しい戦略の概要をプロンプトとして送る
4. AIがコードを生成・修正
5. テスト → 修正 → デプロイ

### 8.2 発表者の活用状況

## 9. Polymarket 5分足市場の特徴

### 9.1 仕組み

- 5分ごとに「この価格を上回るか下回るか」を予測する
- 各5分間の開始時に予測を立て、結果に応じて利益/損失が確定

### 9.2 利点

| 特徴 | 詳細 |
|-----|------|
| **レバレッジなし** | 清算（ロスカット）リスクがない |
| **手数料** | 指値注文なら無料 |
| **最小ベットサイズ** | $1から参加可能 |
| **組み込みレバレッジ** | 5分足市場自体が高リスク・高リターン構造 |

### 9.3 バックテスト用データ

- Coinbaseの1分足データを使用（Polymarket固有のデータは不要）
- BTC/USDの価格データで十分

### 9.4 注意

> 発表者は「5分マーケットは**かなり危険**」と明言している（2,500倍レバレッジに相当する値動き）

## 10. ボットデプロイの具体的手順

### ステップ1: 準備
- バックテストで有効性確認済みの戦略を用意する
- テンプレートとなるボットコード（CVDボット等）を準備する

### ステップ2: 複数アカウント作成
- Polymarketの複数アカウントを作成する
- Polymarketにはサブアカウント機能がないため、独立したアカウントを使用
- 各アカウントにMetaMaskでウォレットを接続

### ステップ3: 資金配分
- 各アカウントに少額（$100程度）を入金
- 1ボット = 1アカウントの原則

### ステップ4: ボット構築
- Claude Codeで既存テンプレートを参照させ、新しい戦略のボットを生成
- 指値注文を使用するよう設定（手数料無料のため）

### ステップ5: 設定確認
- 注文サイズを最小限（$1〜$1.50）に設定
- リスクコントロールが組み込まれているか確認
- デバッグ出力をオフにする

### ステップ6: デプロイ
- VPS（仮想プライベートサーバー）上でボットを起動
- エラーがないか初期確認

### ステップ7: インキュベーション
- 最低数週間〜1ヶ月、小額で稼働させる
- 定期的にログと損益を確認
- 問題がなければ段階的にサイズアップ

## 11. ボット構築時のチェックリスト

- [ ] **リスクコントロール**が組み込まれているか（最重要）
- [ ] **既存ポジションの確認**ロジックがあるか（重複注文の防止）
- [ ] **指値注文**を使用しているか（成行注文の3倍のコスト回避）
- [ ] バックテストした**戦略を正確に実装**しているか
- [ ] エントリー方法を決定しているか（ストップロス or P&Lベース）
- [ ] **自動実行の間隔**を設定しているか（数秒〜数分ごと）
- [ ] **デバッグ出力**を本番環境でオフにしているか
- [ ] **最小サイズ**で開始する設定になっているか

## 12. 成行注文 vs 指値注文 - 手数料の罠

### 12.1 コスト比較

> 成行注文は指値注文の**約3倍のコスト**

### 12.2 具体的な計算例

**条件:**
- 口座残高：$25,000
- レバレッジ：40倍
- 取引頻度：1日5回

**結果:**
| 注文方法 | 口座破綻までの日数 |
|---------|----------------|
| 成行注文 | **31日** |
| 指値注文 | **748日**（+717日） |

### 12.3 なぜ人間は成行注文を使いがちか

- 扁桃体が「今すぐ入りたい、今すぐ出たい」という衝動を引き起こす
- ボットなら忍耐強く指値注文で約定を待てる

### 12.4 Polymarketの手数料

- **指値注文：手数料無料**
- 成行注文：手数料が発生

## 13. アルファ減衰への対策

### 13.1 アルファ減衰とは

全員が同じ戦略・同じボットを使えば、その戦略の利益（アルファ）はゼロに収束する現象。

### 13.2 Jim Simonsの教え

> 「システムを常により良くし続けなければならない。なぜなら、それは他の全員もやっていることだから」
> — Jim Simons（Renaissance Technologies創設者、純資産$310億）

### 13.3 対策

- プラグ＆プレイの「聖杯」ボットは存在しない
- 常に新しいアイデアをリサーチし、バックテストし、改善し続ける
- 自分独自の戦略を構築することが最大のエッジとなる

## 16. 重要な警告・注意事項

### 投資リスク

- **過去の実績は将来の成果を保証しない** — バックテストの限界を常に意識すること
- **5分マーケットは「かなり危険」** — 2,500倍レバレッジ相当の値動き
- レバレッジ取引は資金を全て失うリスクがある

### 技術的リスク

- ボットのバグにより資金を失う可能性がある
- 重複注文の防止ロジックを必ず実装すること
- AIからコードをコピーする際、改行位置にスペースが入りエラーになることがある
- 市場データにはギャップや不正確さがある場合がある

### 戦略的リスク

- アルファ減衰：同じ戦略を使う人が増えれば利益は減少する
- プラグ＆プレイの聖杯は存在しない
- 常にシステムを改善し続ける必要がある

### 取引所の注意

- MEXC（メクシー）は悪い評判が多いため非推奨
- 取引所はカジノと同じ仕組みで、感情を煽るよう設計されている

### 発表者の免責事項

- 動画の後半は有料コース（Algo Trade Camp / moondub.com）への誘導が含まれる
- 発表者自身も複数回の清算を経験しており、自動売買が万能ではないことを示唆している

## 17. まとめ

この動画の核心メッセージは以下の3点に集約される。

### 1. 手動トレードをやめる
感情（扁桃体）は論理の40倍速く発火する。人間が感情的な状態で合理的判断を下すことは神経科学的に不可能であり、レバレッジ取引と組み合わさると致命的な損失につながる。

### 2. RBIシステムに従う
- **Research**: 書籍、ポッドキャスト、Google Scholarから検証済みの戦略を見つける
- **Backtest**: Claude AIを使って過去データで高速検証する
- **Incubate**: 最小額で1ヶ月間テストし、段階的にスケールアップする

### 3. AIをレバレッジとして使う
Claude Codeを使えば、コーディング知識がなくてもバックテストとボット構築が可能。従来8時間以上かかっていたプロセスが1時間に短縮され、5つのボットを2〜3日で構築できる。

> **「コードは偉大な均等化装置（Great Equalizer）だが、AIがコードを誰にでも利用可能にした」** — MoonDev
